## Summary
- add new `throwables()` extensions for `Iterable`, `Sequence`, and `Array` of `AsyncResult<*>`
- deprecate `getAllThrowables()` in favor of `throwables()` while keeping backward compatibility
- add new `incompletes()` extensions for `Iterable`, `Sequence`, and `Array` returning `List<Incomplete>`
- add exhaustive tests for mixed and empty collections across Iterable/Sequence/Array
- update docs collection utilities in `docs/core.md`

## API changes
```kotlin
Iterable<AsyncResult<*>>.throwables(): List<Throwable>
Sequence<AsyncResult<*>>.throwables(): List<Throwable>
Array<out AsyncResult<*>>.throwables(): List<Throwable>

@Deprecated("Use throwables() instead", ReplaceWith("throwables()"))
Iterable<AsyncResult<*>>.getAllThrowables(): List<Throwable>

Iterable<AsyncResult<*>>.incompletes(): List<Incomplete>
Sequence<AsyncResult<*>>.incompletes(): List<Incomplete>
Array<out AsyncResult<*>>.incompletes(): List<Incomplete>
```

## Tests
Added coverage for:
- `throwables()` on Iterable, Sequence, Array
- `incompletes()` on Iterable, Sequence, Array
- deprecated `getAllThrowables()` remains functional
- empty collections
- mixed collections (`Success`, `Error`, `Loading`, `NotStarted`)
